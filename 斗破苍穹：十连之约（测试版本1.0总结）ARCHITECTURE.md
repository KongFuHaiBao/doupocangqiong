## 一、核心数据

### 1.1 关卡 (ALL_STAGES, 约第137行)

`generateStages()` 生成210关，10区域。每关结构：

```js
{ id, region, regionName, stageNumber, name, type: 'normal'|'boss'|'final',
  powerThreshold, goldReward, novelChapter, description }
```

**战力公式**: `Math.round((basePow + increment * i * powMult) * (1 + region * 0.04) / 100) * 100`

**金币公式**: `Math.round((baseGold + goldMult * baseGold * i * 0.5) * (1 + region * 0.4) / 100) * 100`

**章节映射**: `novelChapter` 字段在 generateStages 末尾通过 `chMap` 数组注入，数组格式为 `[[R0章],[R1章],...]`。每个关卡绑定到原著章节号（对应 `<script id="ch_N">` 中的N）。

### 1.2 角色 (ALL_CHARACTERS, 约第227行)

`generateCharacters()` 生成全部角色。关键字段：

```js
{ id, name, rarity: 'N'|'R'|'SR'|'SSR', region, element, basePower,
  isXiaoYan, regionVersion, isRecurring, recurringGroup, description }
```

**SSR权重系统**（约第530行）：抽卡时按权重随机选SSR：
- 萧炎(isXiaoYan): weight=1
- 核心角色(isRecurring): weight=3  
- 普通SSR: weight=6

**核心角色覆盖**（约第568行）：抽到`isRecurring`角色且已有同组旧版时，自动覆盖+继承星数，逻辑同萧炎。

**nBase数组**（决定N/R/SR/SSR战力基数）：`[25, 50, 90, 150, 250, 500, 900, 1600, 2800, 5000]`（按区域索引）

**战力倍数**: N=nBase*0.9, R=nBase*2.5, SR=nBase*6, SSR=nBase*15, XY=nBase*18

### 1.3 斗技 (ALL_TECHNIQUES, 约第323行)

18种斗技，每项：`{ id, name, element, grade, basePower, description }`

### 1.4 丹方 (ALL_RECIPES, 约第360行)

12张丹方，每项：`{ id, name, herbs:[{name,qty}], pillPower, unlockStage, sellPrice }`

### 1.5 商店物品 (ALL_ITEMS, 约第385行)

28种商品，含 药材/魔核/丹药 三类，`permanent:true` 为常驻商品。

## 二、游戏状态 (PlayerData)

存档键 `btth_save_v2`，存储在 `localStorage`。默认状态：

```js
{
  gold: 1000000, currentRegion: 0, currentStage: 0,
  clearedStages: {},       // {stageId: true}
  characters: [],           // [{id, stars(1-5), techniques:[tid|null, tid|null]}]
  team: [],                 // 出战阵容，最多6个角色ID
  bag: { herbs:{}, cores:{}, pills:{} },
  idleType: null|'technique'|'pill', idleTarget: null, idleStartTime: null,
  cultivation: { level: 0, startTime: null },
  pitySR: 0, pitySSR: 0,   // 抽卡保底计数，跨区域不重置
  autoRead: true,           // 过关是否自动弹出小说
  disabledSSRs: {},         // {ssrId: true} 被禁用的SSR
  xiaoYanCurrentVersion: 0, // 萧炎当前区域版本号
  techniques: {}            // {techId: level}
}
```

## 三、核心引擎函数

### 战力计算
- `calcCharPower(charState)` — 单个角色战力：基础×星级倍率 + 斗技加成(匹配100%/不匹配80%) + 萧炎修炼加成
- `getTotalPower()` — 仅统计 `state.team` 中的6个角色战力

### 抽卡引擎
- `gachaPull()` — 单抽：概率→卡池→权重选择→处理获得
- `gachaTenPull()` — 十连抽
- `processCharacterAcquisition(chardata)` — 新角色/重复升星/满星返金币/核心覆盖

### 战斗
- `battleStage(stageId)` — 战力对比，通关后返回 `{success, power, threshold, gold, stage}`
- `advanceRegion()` — 进入下一区域：萧炎进化、SSR禁用、队伍自动补满6人
- `advanceToNextRegion()` — 带弹窗提示卡池更新

### 挂机
- `getIdleProgress()` — 计算斗技/炼丹挂机进度（基于时间戳差值，最大12小时）
- `getCultivationProgress()` — 萧炎修炼进度（独立槽位）

### 存档
- `loadGame()` / `saveGame()` — localStorage读写
- `exportSave()` / `importSave(file)` — JSON文件导出/导入

## 四、UI渲染函数

### 导航
- `switchTab(tab)` — 切换7个标签：main/gacha/char/bag/idle/sxf/more
- `renderTab(tab)` — 根据tab名分发到对应渲染函数

### 各标签
| 函数 | 说明 |
|------|------|
| `renderMain()` → `renderRegions()` / `renderStages()` | 主线爬塔，区域→关卡两级 |
| `renderGacha()` | 抽卡界面，含当前SSR池展示 |
| `renderChar()` → `renderTeamView()` / `renderCharDetail()` | 角色列表/阵容/详情 |
| `renderBag()` | 背包三分类 |
| `renderIdle()` | 挂机控制 |
| `renderSxf()` → `renderSxfChapters()` | 小说标签：区域→章节 |
| `renderMore()` → 修炼/SSR图鉴/商店/存档 | 更多面板 |
| `showModal(title,content)` / `closeModal()` | 通用弹窗 |

### 列表排序
- `charSort` 变量：`'rarity'` | `'power'`，角色列表和阵容界面共用
- `charView` 变量：`'list'` | `'team'` | `'detail'`

## 五、内置阅读器

### 数据
- 章节数据在 `<script id="ch_0">` 到 `<script id="ch_1522">`，`type="text/plain"` 不执行
- `TITLES` 数组（约23300字符）：1523章的标题
- `R_TOTAL = 1523`

### UI
- `#reader-container` 全屏div，默认 `display:none`
- 头部：字体缩放(A-/A+)、翻页/滚动切换、退出✕
- 底部：上一章/下一章（仅滚动模式）+ 可关闭自动跳转提示
- `#r-cb` 内容区，`#r-page-ind` 页码指示器（仅翻页模式）

### JS函数
| 函数 | 说明 |
|------|------|
| `readerOpen(startCh, endCh)` | 打开阅读器，endCh为0则无自动退出 |
| `readerExit()` | 关闭阅读器，保存翻页/字体设置，恢复小说tab视图 |
| `readerLoad(i)` | 加载第i章（从`ch_i`标签读数据→渲染到#r-cb） |
| `buildPages()` | 翻页模式：用隐藏div精确测量每段高度，分段为页 |
| `showPage()` | 显示当前页 |
| `readerNextPage()` / `readerPrevPage()` | 翻页模式翻页，末页自动跳下一章 |
| `readerNext()` / `readerPrev()` | 滚动模式上下章 |
| `readerSetFont(d)` | 字体缩放，翻页模式下重建页面 |
| `readerTogglePage()` | 滚动↔翻页模式切换 |
| `r_pageMode` / `r_fsIdx` | 当前模式/字体索引 |
| `r_savedMode` / `r_savedFont` | 跨章节保留的设置 |

### 阅读器入口
1. **主线过关**：`state.autoRead===true` 时自动调用 `readerOpen(章节号, 下一关章节号)`
2. **已通关关卡**：点击已通关关卡同样弹小说（不给奖励）
3. **小说标签**：`sxfView='regions'→'chapters'→'reader'` 三级导航，`sxfScrollPos={区域:位置}` 独立记忆

## 六、修改常见需求的指引

### 改战力/平衡
- nBase数组（约第305行附近，共3处）：改[N,R,SR生成中的基数]
- SSR硬编码值：用Python脚本批量替换 `basePower:数字`
- 关卡公式：改 `(1+c.r*0.04)` 和 `powMult` (1/1.5/2.5)

- 直接修改 `generateCharacters()` 中的 `chars.push(...)` 行
- SSR池由 `region` 字段自动分组

### 改抽卡概率
- `GACHA_PROBS` 变量（约第130行）
- 权重在 `gachaPull()` 函数中（约第530行）

### 改关卡名/章节映射
- `chMap` 数组在 `generateStages()` 末尾（约第222行）
